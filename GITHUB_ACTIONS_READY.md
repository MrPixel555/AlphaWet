# GitHub Actions readiness notes

This repository is prepared for Android APK builds in GitHub Actions without committing a persistent `android/` directory.

## What changed

- The CI workflow now delegates Android project preparation to `tool/ci_prepare_android_project.sh`.
- `tool/prepare_android_runtime.sh` no longer assumes a hardcoded Android package.
- The Kotlin overlay package is rewritten to the package generated by `flutter create` in CI.
- CI verifies whether real Xray binaries are present in `assets/xray/android/...` and emits warnings instead of failing the build when placeholders are still present.
- Optional secure-import `--dart-define` values can be injected from GitHub Secrets.
- On CI failure, the prepared Android project is uploaded as an artifact for debugging.

## Recommended GitHub Secrets

- `AW_TRANSPORT_KEY_BASE64`
- `AW_ED25519_PUBLIC_KEY_BASE64`
- `AW_ALLOW_LEGACY_PLAINTEXT_IMPORT`

## Real Xray runtime packaging

For runtime launch to work on Android devices, package the extracted binaries at:

- `assets/xray/android/arm64-v8a/xray`
- `assets/xray/android/x86_64/xray`

The APK can still compile if these are not present, but runtime Xray launch will remain unavailable.
